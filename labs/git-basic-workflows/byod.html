Using Git to Copy Code and Setting Up the Local Environment

Now that your workstation is ready to go, the next step is to retrieve the code and install the lab specific requirements.

**Note: You only need to do this setup process once. If you have previously cloned the repository and created your virtual environment, you do not need to do it again for this lab.**

  1. The code for this lab is available on GitHub in the appropriate repository for your event:

    • DNA: https://github.com/CiscoDevNet/dnav3-code
    • Security: https://github.com/CiscoDevNet/dne-security-code
    • Data Center Infrastructure: https://github.com/CiscoDevNet/dciv2-code
    • Meraki: https://github.com/CiscoDevNet/meraki-code

  2. Open a bash terminal and change to the directory where you would like to clone the repository. For example, a directory called code/ under your $HOME directory.

                
cd ~/code

  3. Clone the repository and change into the new folder. This example is for dnav3-code, but you can also substitute dne-security-code or dciv2-code or meraki-code as needed.

                
git clone https://github.com/CiscoDevNet/dnav3-code
cd dnav3-code

  4. Create a Python 3 virtual environment and activate it.

                
# MacOS or Linux
python3 -m venv venv
source venv/bin/activate
            
        
                
# Windows git-bash
py -3 -m venv venv
source venv/Scripts/activate
            
          
                  
  # Windows PowerShell (Note: not used at DevNet Express.)
  py -3 -m venv venv
  .venv\Scripts\activate

Alright, you should be all set and ready to start the lab!
